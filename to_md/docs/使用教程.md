# wx2md.exe 使用教程

> 面向**普通用户**:不需要懂 Python,不需要装任何环境,**双击/命令行直接运行 `wx2md.exe`** 就能把微信公众号文章保存成本地可离线阅读的 Markdown + HTML + 图片。

## 1. 这是什么

`wx2md.exe` 是一个**绿色单文件**命令行工具,功能:

- 输入:一个或多个微信公众号文章链接(`https://mp.weixin.qq.com/s/xxx`)
- 输出:每篇文章生成一个文件夹,内含
  - `article.md` —— Markdown 文本(可导入 Obsidian / Typora / Logseq)
  - `article.html` —— 离线 HTML(浏览器双击直接看,样式接近公众号原版)
  - `images/` —— 文章里**所有图片**已下载到本地

特点:文章并发 + 图片并发,几十篇文章几分钟搞定;支持 URL 列表批量。

## 2. 准备工作

### 2.1 拿到 `wx2md.exe`

把 `wx2md.exe` 放到任意一个**英文路径**的文件夹下,例如:

```
D:\tools\wx2md\wx2md.exe
```

> 路径里**尽量不要有空格和中文**,可以省掉很多麻烦。

### 2.2 打开命令行(PowerShell)

在 `wx2md.exe` 所在文件夹的**地址栏**里输入 `powershell` 回车,会直接在该目录打开 PowerShell。

或者:任意位置打开 PowerShell,然后:

```powershell
cd D:\tools\wx2md
```

### 2.3 验证能跑

```powershell
.\wx2md.exe -V
```

看到类似 `wx2md 0.1.0` 即正常。

> Windows 上输入 `.\` 是必须的,告诉 PowerShell "运行当前目录下的 exe"。

## 3. 第一次运行:抓一篇文章

### 3.1 复制一个公众号文章链接

在微信里打开任意一篇文章 → 右上角 `···` → "复制链接",得到形如:

```
https://mp.weixin.qq.com/s/oKi8k-GTyL6ggSo4rqY3hw
```

### 3.2 跑

```powershell
.\wx2md.exe https://mp.weixin.qq.com/s/oKi8k-GTyL6ggSo4rqY3hw
```

终端会输出:

```
19:32:01 [INFO] fetch https://mp.weixin.qq.com/s/...
19:32:02 [INFO] [文章标题] 12 images
19:32:04 [INFO] done 文章标题 -> output\文章标题
19:32:04 [INFO] finished: 1 ok / 0 failed / 1 total
```

### 3.3 查看产物

默认输出到当前目录的 `output\` 文件夹下:

```
output\
└── 文章标题\
    ├── article.md
    ├── article.html       ← 双击这个,浏览器看效果
    └── images\
```

双击 `article.html` 就能在浏览器里离线阅读,所有图片都已经本地化。

## 4. 批量抓取:一次抓很多篇

### 4.1 准备一个 URL 列表

新建一个 `urls.txt`(放在 `wx2md.exe` 同目录),每行一个链接,`#` 开头是注释:

```text
# 我的收藏
https://mp.weixin.qq.com/s/oKi8k-GTyL6ggSo4rqY3hw
https://mp.weixin.qq.com/s/NcQmDNnuMaH96qMBuENoNg

# 下面这条暂时不抓
# https://mp.weixin.qq.com/s/skip-this-one
```

> 本教程附带了一个现成的样例文件,见下方 [测试数据](#7-测试数据)。

### 4.2 跑

```powershell
.\wx2md.exe -i urls.txt -o output -w 4 -t 8
```

意思:

| 参数 | 含义 |
|------|------|
| `-i urls.txt` | 从 `urls.txt` 读 URL 列表 |
| `-o output` | 结果输出到 `output\` 目录 |
| `-w 4` | **4 篇文章并行**抓取 |
| `-t 8` | 每篇文章内 **8 张图片并行**下载 |

### 4.3 性能建议

| 场景 | 推荐 |
|------|------|
| 几篇文章、图特别多 | `-w 1 -t 16` |
| 几十篇、图少 | `-w 8 -t 4` |
| 网络一般 / 怕被限速 | `-w 2 -t 4` |
| 不知道选啥就用默认 | 不加 `-w -t`,默认 4×8 |

并发不是越大越好;`-w` 超过 8 容易被微信侧限速。

## 5. 常用参数速查

```
.\wx2md.exe [选项] [URL...]

位置参数:
  urls                 一个或多个文章 URL,可同时跟多个

可选参数:
  -i, --input 文件     URL 列表文件(每行一个,# 注释)
  -o, --output 目录    输出根目录(默认 output)
  -w, --workers N      文章级并发数(默认 4)
  -t, --img-workers N  每篇文章内图片并发数(默认 8)
  -v, --verbose        详细日志
  --debug              调试模式:完整报错 + 失败诊断
  -V, --version        显示版本
  -h, --help           显示帮助
```

### 几个常见用法

抓一篇:

```powershell
.\wx2md.exe https://mp.weixin.qq.com/s/xxx
```

同时抓多篇(URL 直接跟在后面):

```powershell
.\wx2md.exe https://mp.weixin.qq.com/s/aaa https://mp.weixin.qq.com/s/bbb
```

指定输出到桌面:

```powershell
.\wx2md.exe -o "$env:USERPROFILE\Desktop\wx文章" -i urls.txt
```

从列表抓 + 输出排错信息:

```powershell
.\wx2md.exe --debug -i urls.txt -o output
```

## 6. 输出文件说明

```
output\
├── 文章标题A\
│   ├── article.md        ← Markdown,引用 images/ 下的图
│   ├── article.html      ← 离线 HTML,样式接近原文
│   └── images\
│       ├── 7f3a1c9b2e4d5601.jpg
│       └── a1b2c3d4e5f60718.png
├── 文章标题B\
│   └── ...
└── _debug\               ← 只在 --debug 模式且有失败时出现
    ├── failed.txt        ← 每行:失败的 URL + 错误消息
    └── report.txt        ← 每条失败的完整报错堆栈
```

要点:

- 图片名 = 内容 sha1 前 16 位 + 扩展名,**重复内容自动去重**;重跑也不会重复下载
- 文章标题里的 `\/:*?"<>|` 等非法字符自动换成 `_`,长度上限 120
- `article.md` 顶部带 YAML frontmatter(标题/账号/作者/日期/原文链接),Obsidian 等工具能识别

## 7. 测试数据

仓库 `examples\` 目录下提供了几份现成的 URL 列表,用来快速验证 `wx2md.exe` 工作正常。

把它们和 `wx2md.exe` 放到同一目录,或者用 `-i` 指定相对/绝对路径都行。

### 7.1 单篇测试(最小用例)

文件:`examples\test_single.txt`

```
# 单篇测试:1 个 URL,跑通基本流程
https://mp.weixin.qq.com/s/oKi8k-GTyL6ggSo4rqY3hw
```

用法:

```powershell
.\wx2md.exe -i examples\test_single.txt -o output\test_single
```

预期:`output\test_single\` 下出现 1 个文章文件夹,日志末尾 `1 ok / 0 failed`。

### 7.2 批量测试(并发用例)

文件:`examples\test_batch.txt`

```
# 批量测试:多个 URL,验证文章级并发
https://mp.weixin.qq.com/s/oKi8k-GTyL6ggSo4rqY3hw
https://mp.weixin.qq.com/s/NcQmDNnuMaH96qMBuENoNg
```

用法:

```powershell
.\wx2md.exe -i examples\test_batch.txt -o output\test_batch -w 2 -t 8
```

预期:并行抓取,`output\test_batch\` 下出现多个文章文件夹。

### 7.3 容错测试(故意混入坏 URL)

文件:`examples\test_invalid.txt`

```
# 容错测试:含一个无效 URL,验证单篇失败不影响其他
https://mp.weixin.qq.com/s/oKi8k-GTyL6ggSo4rqY3hw
https://mp.weixin.qq.com/s/THIS_URL_DOES_NOT_EXIST_XYZ
```

用法:

```powershell
.\wx2md.exe --debug -i examples\test_invalid.txt -o output\test_invalid
```

预期:

- 正常 URL 成功,无效 URL 失败
- 日志显示 `1 ok / 1 failed / 2 total`
- `output\test_invalid\_debug\failed.txt` 和 `report.txt` 内有失败详情

## 8. 常见问题

### Q1:运行报"无法将 wx2md.exe 项识别为 cmdlet..."

PowerShell 不会从当前目录找命令,要加 `.\`:

```powershell
.\wx2md.exe -V
```

### Q2:Windows Defender / 浏览器下载时提示"未签名/有风险"

`wx2md.exe` 是用 PyInstaller 打包的 Python 程序,没买代码签名证书,首次会被提示。
可以选择"仍要运行"。介意的话用源码版:见仓库 `README.md` 的"开发指南"。

### Q3:中文文件夹名乱码

PowerShell 改成 UTF-8 代码页:

```powershell
chcp 65001
```

或换用 Windows Terminal。

### Q4:`content div #js_content not found`

可能原因:文章已删 / 文章设置了"登录可见" / 网络代理拦截。
浏览器打开同一 URL 验证一下,或加 `--debug` 看完整响应。

### Q5:图片下载很多失败 / 全部 403

工具已经默认带了微信防盗链需要的 `Referer`,正常情况不会发生。如果出现:

- 多半是 IP 被短时限速,**降低并发**:`-w 2 -t 4`
- 个别老图片地址确实失效了,会跳过,不影响其他

### Q6:想把所有 MD 导入 Obsidian

直接把 `output\` 整个文件夹拖进 Obsidian 当 vault 即可。
`article.md` 用相对路径 `images/xxx.jpg` 引用图片,Obsidian 能识别。

### Q7:重跑同一个 URL 会重复下载吗

不会。图片按内容 hash 命名,文件存在就跳过;文章 MD/HTML 会覆盖更新。

### Q8:能不能抓"分享给朋友才能看"的文章

不能。这类文章服务端不返回正文 HTML,本工具会报 `#js_content not found`。

## 附:一行流命令(熟手向)

抓一篇:

```powershell
.\wx2md.exe https://mp.weixin.qq.com/s/xxx
```

抓列表 + 排错:

```powershell
.\wx2md.exe --debug -i urls.txt -o output -w 4 -t 8
```

加入 PATH 后任意目录直接用:

```powershell
# 把 wx2md.exe 所在目录加入用户 PATH,以后可以省掉 .\
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";D:\tools\wx2md", "User")
# 重开终端后:
wx2md -V
```
