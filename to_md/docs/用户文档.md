# wx2md 用户文档

把微信公众号文章（`https://mp.weixin.qq.com/s/...`）一键转成本地的 **Markdown + HTML + 图片**，离线可读、可导入 Obsidian / Typora / Logseq。

本文档面向**最终用户**，只讲怎么用。安装、原理、开发详见 [`README.md`](../README.md)。

## 1. 我能用它做什么

| 输入 | 输出 |
|------|------|
| 1 个公众号文章 URL | 1 个文件夹（`article.md` + `article.html` + `images/`） |
| N 个 URL（命令行 / 文本文件） | N 个文件夹，并发抓取 |
| 含坏 URL 的列表 | 跳过坏的、其余照抓，可选 `--debug` 落盘失败原因 |

特点：

- **绕过防盗链** —— 图片自动加 `Referer` 下载到本地，不再热链微信 CDN
- **内容去重** —— 图片以 sha1 命名，重复内容只存一份，重跑也不会重复下载
- **保留排版** —— HTML 输出尽量贴近公众号原样式
- **YAML frontmatter** —— `article.md` 顶部带标题/作者/日期/原文链接，Obsidian 友好

## 2. 运行方式速选

按场景挑一种，命令换掉前缀即可，参数完全一样。

| 场景 | 命令前缀 | 准备工作 |
|------|----------|----------|
| 开发期 / 本地仓库 | `uv run wx2md` | `uv sync` |
| 已激活 .venv | `python -m wx2md` | `.venv\Scripts\Activate.ps1` |
| 全局命令 | `wx2md` | `uv tool install .` |
| 绿色 exe（不装 Python） | `.\wx2md.exe` | 把 `wx2md.exe` 放到一个英文路径 |

下文示例统一用 `wx2md` 表示，请按你的方式替换。

## 3. 单篇转换

### 3.1 最简形态

```powershell
wx2md https://mp.weixin.qq.com/s/oKi8k-GTyL6ggSo4rqY3hw
```

终端输出大致：

```
19:32:01 [INFO] fetch https://mp.weixin.qq.com/s/...
19:32:02 [INFO] [文章标题] 12 images
19:32:04 [INFO] done 文章标题 -> output\文章标题
19:32:04 [INFO] finished: 1 ok / 0 failed / 1 total
```

产物：

```
output\文章标题\
├── article.md
├── article.html      ← 双击在浏览器查看
└── images\
```

### 3.2 一次给多个 URL

直接在命令行后面跟多个，**用空格隔开**：

```powershell
wx2md https://mp.weixin.qq.com/s/aaa https://mp.weixin.qq.com/s/bbb
```

默认 `-w 4`，会并行抓取。

### 3.3 输出到指定目录

用 `-o`：

```powershell
wx2md -o "D:\我的文章" https://mp.weixin.qq.com/s/xxx
```

桌面版（PowerShell 环境变量）：

```powershell
wx2md -o "$env:USERPROFILE\Desktop\wx文章" https://mp.weixin.qq.com/s/xxx
```

## 4. 批量转换

### 4.1 准备 URL 列表

新建 `urls.txt`，每行一个 URL，`#` 开头是注释，空行忽略：

```text
# 收藏夹 - 技术
https://mp.weixin.qq.com/s/oKi8k-GTyL6ggSo4rqY3hw
https://mp.weixin.qq.com/s/NcQmDNnuMaH96qMBuENoNg

# 这条暂时不抓
# https://mp.weixin.qq.com/s/skip-me
```

仓库 `examples\` 下已有现成样例：

- `examples\test_single.txt` —— 单篇，跑通最小流程
- `examples\test_batch.txt` —— 多篇，验证并发
- `examples\test_invalid.txt` —— 故意混坏 URL，验证容错

### 4.2 批量跑

```powershell
wx2md -i urls.txt -o output -w 4 -t 8
```

| 参数 | 含义 |
|------|------|
| `-i urls.txt` | 从 `urls.txt` 读 URL 列表 |
| `-o output` | 输出到 `output\` |
| `-w 4` | 同时处理 4 篇文章 |
| `-t 8` | 每篇文章内同时下载 8 张图片 |

理论并发 = `-w × -t` = 32 个图片下载并行。所有线程共用一个 HTTP 连接池。

### 4.3 列表 + 命令行 URL 混用

可以同时给 `-i` 和位置参数，会**合并后自动去重**：

```powershell
wx2md -i urls.txt https://mp.weixin.qq.com/s/extra1 https://mp.weixin.qq.com/s/extra2
```

### 4.4 并发调优参考

| 场景 | 推荐 |
|------|------|
| 几篇文章，图特别多 | `-w 1 -t 16` |
| 几十篇，图少 | `-w 8 -t 4` |
| 网络一般 / 怕被限速 | `-w 2 -t 4` |
| 不确定 | 默认（`-w 4 -t 8`） |

`-w` 不建议超过 8，微信侧容易触发限速。

## 5. 参数详解

```
wx2md [-h] [-i INPUT] [-o OUTPUT] [-w N] [-t N] [-v] [--debug] [-V] [URL ...]
```

### 位置参数

| 参数 | 说明 |
|------|------|
| `URL ...` | 0 个或多个公众号文章链接。0 个时必须配合 `-i`。 |

### 可选参数

| 参数 | 默认 | 说明 |
|------|------|------|
| `-i, --input 文件` | 无 | URL 列表文件，每行一个，`#` 注释，空行忽略 |
| `-o, --output 目录` | `output` | 输出根目录，不存在会自动创建 |
| `-w, --workers N` | `4` | **文章级**并发数：同时处理几篇文章 |
| `-t, --img-workers N` | `8` | **图片级**并发数：每篇文章内同时下载几张图 |
| `-v, --verbose` | 关 | DEBUG 级日志（更啰嗦） |
| `--debug` | 关 | 调试模式：完整 traceback + HTTP 日志 + 失败 dump（自动包含 `-v`） |
| `-V, --version` | — | 显示版本号 |
| `-h, --help` | — | 显示帮助 |

### 参数选择心得

- **`-w` vs `-t`**：`-w` 决定有多少篇文章在跑，`-t` 决定每篇文章内部图片下载多并行。图多用 `-t` 拉满，图少多文章用 `-w` 拉满。
- **`-o`**：每次跑可以指向不同子目录，把不同主题分开归档（如 `-o output\技术`、`-o output\新闻`）。
- **`-v` vs `--debug`**：日常看进度用 `-v`；排错、记录失败用 `--debug`。

## 6. 输出结构

```
output\
├── 文章标题A\
│   ├── article.md          ← Markdown，YAML frontmatter + 相对路径图
│   ├── article.html        ← 离线 HTML，浏览器双击可看
│   └── images\
│       ├── 7f3a1c9b2e4d5601.jpg
│       └── a1b2c3d4e5f60718.png
├── 文章标题B\
│   └── ...
└── _debug\                 ← 仅 --debug 且有失败时出现
    ├── failed.txt          ← URL<TAB>错误概要
    └── report.txt          ← 完整 traceback
```

要点：

- 图片名 = `sha1(内容)[:16] + 扩展名` → 同一张图跨文章只存一份；重跑跳过已存在的文件
- 标题里的 `\ / : * ? " < > |` 被替换为 `_`，长度截到 120 字符（Windows 文件名安全）
- `article.md` 的 frontmatter 示例：

```yaml
---
title: 文章标题
account: 公众号名
author: 作者
date: 2026-06-10
url: https://mp.weixin.qq.com/s/xxx
---
```

## 7. 调试与排错

### 7.1 加 `--debug` 看完整信息

```powershell
wx2md --debug -i urls.txt -o output
```

- 终端会打出每次失败的完整 Python traceback
- `httpx` 的请求/响应明细也会输出，对排查 403/302 很有用
- 失败的 URL 会汇总到 `output\_debug\failed.txt` 与 `report.txt`

### 7.2 常见报错

| 报错 | 含义 | 对策 |
|------|------|------|
| `content div #js_content not found` | 文章已删、登录可见、反爬验证 | 浏览器打开同 URL 验证 |
| 大量图片 `403` | 被微信限速 | 降低 `-w` 和 `-t`（如 `-w 2 -t 4`） |
| `httpx.ConnectTimeout` | 网络慢 | 重试，或检查代理 |
| `无法识别 wx2md.exe` (Windows) | PowerShell 不查当前目录 | 用 `.\wx2md.exe` |

### 7.3 退出码

- `0`：全部成功
- `1`：至少 1 篇失败

可在 CI / 脚本里用退出码判断是否需要告警。

## 8. 常用配方

抓单篇：

```powershell
wx2md https://mp.weixin.qq.com/s/xxx
```

一次抓几个：

```powershell
wx2md https://mp.weixin.qq.com/s/aaa https://mp.weixin.qq.com/s/bbb
```

从文件批量 + 自定义并发：

```powershell
wx2md -i urls.txt -o output -w 4 -t 8
```

批量 + 排错信息落盘：

```powershell
wx2md --debug -i urls.txt -o output
```

输出到桌面：

```powershell
wx2md -o "$env:USERPROFILE\Desktop\wx文章" -i urls.txt
```

低并发抗限速：

```powershell
wx2md -i urls.txt -w 2 -t 4
```

用样例数据快速验证安装：

```powershell
wx2md -i examples\test_single.txt -o output\test
```
